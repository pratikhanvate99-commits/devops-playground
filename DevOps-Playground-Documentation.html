01 — Overview

What Is This Project?

The DevOps Playground is a fully functional, locally deployable DevOps portfolio project that simulates an enterprise-grade microservices environment. It is designed to demonstrate real-world DevOps skills including containerization, Infrastructure as Code, observability, self-healing, fault injection, and CI/CD automation — all running on a single Windows machine with Docker Desktop.

The project is ideal for DevOps Engineers, SREs, and Cloud Engineers to showcase hands-on experience to recruiters and interviewers. Every component is built to reflect production-level patterns without requiring expensive cloud subscriptions.

Core Philosophy

Rather than deploying a simple web application, this project builds a complete platform — a microservices architecture surrounded by monitoring, reliability testing, automated delivery, and a rich operations dashboard. It answers the key DevOps engineering interview question: "How would you ensure your system is observable, resilient, and automatically recoverable?"

04 — Features

Complete Feature Set

📊 Control Center Dashboard

  • Live service health cards for API Gateway, PostgreSQL, and Worker Service — updated every 2 seconds
  • Real-time Gateway Memory (RSS) usage display in megabytes
  • SVG Sparkline charts showing live API request throughput and end-to-end latency trends
  • Scrollable event console with timestamped logs for service recovery, chaos toggles, and task events

🔥 Chaos Engineering Laboratory (6 Experiments)

⏱ Network Latency Injection

Adds a flat 2-second delay to all API gateway routes to simulate network congestion or packet loss.

💥 HTTP 500 Error Rate

Causes 50% of incoming API requests to randomly fail with a 500 Internal Server Error status.

🧠 Memory Leak Simulator

Allocates 5MB to heap every 100ms inside Node.js, rapidly consuming RAM until Docker OOM kills it.

🔥 Worker CPU Stress

Forces a Python thread into a math busy-loop for 20 seconds, pegging a full CPU core to 100%.

🔐 Auth Token Failure

Returns 401 Unauthorized on 30% of requests to simulate expired or corrupted JWT tokens.

🗄 Database Query Bottleneck

Injects a 3-second delay directly inside all PostgreSQL query transactions to test DB timeout handling.

Crash Buttons: Kill the API Gateway or Worker container process entirely (exit code 1), triggering Docker's restart = always self-healing routine in real-time.

💾 Database Sandbox

  • Create named task records directly to the PostgreSQL database via the React form
  • Python Worker Service polls the task queue every second and updates status to "processing" → "completed"
  • Live paginated task log with color-coded status badges: Pending / Processing / Completed
  • Demonstrates the full write-to-db → async-processing → state-update pipeline

🖥 Simulated CI/CD Pipeline Runner

  • Click "Trigger Pipeline Rollout" to simulate a full GitHub Actions deployment workflow
  • 5-stage animated progress bars: Checkout → Lint → Docker Build → Trivy Scan → Registry Push
  • Real-time terminal console printing timestamped build logs for each stage
  • Shows how CI/CD quality gates prevent bad images from reaching production

📡 Observability & Monitoring Suite

  • Grafana auto-provisions Prometheus datasource and DevOps Playground dashboard on first boot
  • Dashboard panels: DB Connection Status, Task Count, Chaos Mode State, API Gateway Throughput, Response Latency, Container CPU and Memory
  • Prometheus scrapes metrics from 3 targets: Gateway, Worker, cAdvisor
  • All raw metrics viewable at http://localhost:5000/metrics
05 — Key DevOps Concepts Demonstrated

DevOps Skills Showcased

Skill Area How It's Demonstrated
Infrastructure as Code All 9 containers, networks, volumes, health checks, and env variables are defined in terraform/main.tf. Zero manual docker commands needed.
Containerization Each service has a dedicated, multi-stage Dockerfile. The frontend uses Node build + Nginx runtime stages for minimal image size.
Self-Healing Architecture Containers use restart = "always". When a crash button kills the process, Docker detects the exit code and automatically boots a fresh container.
Observability (Metrics) Custom Prometheus metrics (counters, histograms, gauges) are instrumented in the Node.js and Python code with meaningful labels and business context.
Chaos Engineering 6 fault injection experiments validate the system's resilience to latency, error rates, memory pressure, CPU spikes, auth failures, and DB bottlenecks.
CI/CD Automation GitHub Actions workflow automates linting (ESLint, flake8), Docker image builds, security scanning (Trivy), and simulates push-to-registry and GitOps deployment.
Security Scanning Trivy scans container images for OS and library CVEs during every CI run. Zero critical vulnerabilities in base images.
Service Mesh Patterns API Gateway pattern routes all external traffic, decoupling the frontend from backend services. All inter-service communication is on a private Docker network.
Asynchronous Processing The Python Worker runs an independent background thread that polls PostgreSQL for pending work items and processes them without blocking the main API thread.
